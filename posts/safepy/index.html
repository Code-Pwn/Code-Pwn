Solution

The main take away from the dockerfile is the location of a “flag” file in the root of the file system/. However, for that to be relevant, code execution is needed.

Analysis of main.py

This script uses the Sympy module and in particular, the parse_expr() function. There is also a note via comment about a stack overflow thread where a comment is made that simpify() uses eval(). eval() is a builtin python function which takes an expression as input and the returns it’s result. If an attacker is able to control the values pased to eval(), they maybe able to achieve code execution. Therefore, on Stack Overflow, they parse_expr() function is recommended for use instead.

However, an inspection of the sympy module’s source code reveles that parse_expr() also uses eval(). Source Code Link

Clearly, just because something is said on Stack Overflow, doesn’t make it true.

Flag

eval() evaluates expressions so the exploit also needs to be an expression. Fortunately, there are several resources that can be used to find an example, including Hacktricks.xyz. One of the provided examples can then be modified to read the flag from the location noted earlier.
